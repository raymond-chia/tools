# 遊戲設計機制文檔

## 核心願景

### 你希望遊戲的核心樂趣是什麼？

- 規劃戰術, 逐一嘗試戰術是否有效。最終發現戰術能穩定成功時會有成就感
- 需要有某種程度的隨機性, 以免像是在下棋, 並產生最優解
- 不能隨機出現敵人, 以免難以設計戰術
- 需要根據喜好戰術培養角色, 讓不同角色使用不同技能合作達成戰術

- 比方來說 pathfinder wrath of the righteous 的 through the ashes DLC 中, 會需要利用地形甚至機關來對付敵人
- 需要針對性培養角色, 才好應付無法依靠地利對付的敵人

### 你希望玩家在戰鬥中主要做什麼決策？

- 主要是根據戰術施放技能, 一步一步完成戰術
- 戰術需要考量資源分配, 比如法術會消耗 MP
- 戰術可能包含位置, 比如把敵人推下懸崖, 或者堵住隘口讓後排盡情輸出
- 重要元素: 位置（光、掩體、高地命中、夾擊、line of sight）、位移（製造地形、利用地形、元素反應）、反應動作、壓力機制、fog of war

## 回合定義

- 單位行動 到 下次單位行動前
- 單位回合開始時檢查 buff / debuff duration != 0
- 單位回合結束才減少 duration, 處理 damage over time

## 遊戲定位

**類型**：戰術規劃 + 試錯 + 協同戰鬥 + 地形利用的回合制戰棋 RPG

**參考遊戲**：

- Pathfinder: Wrath of the Righteous (地形與機關戰術)
- Divinity: Original Sin 2 (元素互動與地形利用)
- Into the Breach (完美信息與位置戰術)
- XCOM 2 (資源管理與風險評估)
- 火焰之紋章 (地形系統與角色養成)

**核心循環**：

1. 觀察戰場（敵人位置、地形、可用資源）
2. 規劃戰術（技能組合、位置利用、資源分配）
3. 執行戰術（移動、施法、團隊合作、利用地形）
4. 評估結果（成功 → 下一關 / 失敗 → 調整戰術重試）

**核心理念**：

> **針對性的策略應該能克服隨機性。**
>
> 隨機性只是讓高風險高報酬的戰術不易實現，但不會完全不可能實現。
> 玩家可以針對性地通過裝備、技能、位置等方式提高成功率，將「賭博式戰術」變成「高成功率戰術」。

## 當前設計問題診斷

### 缺失的關鍵要素

#### 1. 地形互動系統不足

**問題**：

- 雖然有 `Shove` 效果（推擊），但沒有**深淵**等致命地形
- 沒有**阻擋視線**的地形（掩體、牆壁）
- 沒有**可互動物件**（機關、陷阱、可破壞物）

**影響**：

- 無法實現「推下懸崖」、「堵住隘口」等戰術
- 缺少地形帶來的戰術深度

#### 2. 技能協同機制不明顯

**問題**：

- 沒有「狀態 + 技能」的聯動（例如：油地 + 火球 = 範圍燃燒）
- 沒有「角色 + 角色」的配合（例如：隊友包夾 → 夾擊傷害加成）

**影響**：

- 無法實現「角色使用不同技能合作達成戰術」
- 技能只是單獨使用，缺乏協同樂趣

#### 3. 資源管理層次不足

**問題**：

- MP 目前只是每場戰鬥的資源
- 沒有跨戰鬥的資源管理（MP 應該是一天的限制）
- 沒有消耗品系統（藥水、卷軸）

**影響**：

- 資源管理決策過於簡單
- 缺少「謹慎使用大招」的緊張感
- 無法實現「連續多場戰鬥」的關卡設計

## 核心機制設計

### 機制 1：地形系統

#### 地形戰術範例

**戰術 1：推下懸崖**

```
佈局：
  敵人 站在 Cliff（懸崖邊）
  Cliff 旁邊是 Pit（深淵）

執行：
  1. 戰士移動到敵人旁邊
  2. 使用「Shove」技能推擊
  3. 敵人被推入 Pit
  4. 掉落即死

結果：
  - 即使是高血量 Boss，也能一擊必殺
  - 風險：需要敵人站在懸崖邊
```

**戰術 2：堵住隘口**

```
佈局：
  山谷地形，只有一條通道（兩側是 Wall）

執行：
  1. 坦克站在通道口
  2. 敵人無法繞過（Wall 不可通行）
  3. 後排安全輸出

結果：
  - 以少敵多的關鍵戰術
  - 地形成為戰術的一部分
```

---

#### 「針對性策略克服隨機性」範例

**情境**：Boss 戰，Boss 有高閃避（evasion = 50）

```
策略 A：直接攻擊（隨機）
    使用「重擊」(accuracy = 30)
    hit_score = 30 + random(1-100) = 31-130
    evade_score = hit_score - 50 = -19 到 80
    → 成功率約 60%（不穩定）

策略 B：針對性策略（克服隨機性）
    1. 使用「祝福術」給自己 +20 accuracy
    2. 裝備「精準項鍊」+10 accuracy
    3. 利用「高地優勢」+10 accuracy（未來擴充）

    現在 accuracy = 70
    evade_score = 70 + random(1-100) - 50 = 21-120
    → 成功率約 95%（穩定！）

策略 C：放棄攻擊，利用地形（完全克服隨機性）
    1. 把 Boss 引到懸崖邊
    2. 使用「Shove」推下去
    → 成功率 100%（如果能引到懸崖邊）
```

**關鍵**：

- ✅ 穩健戰術必定存在（策略 B、C）
- ✅ 玩家可以選擇承受多少風險
- ✅ 針對性準備能克服隨機性

### 機制 3：資源管理系統（MP 是一天的限制）

#### 設計目標

- MP 是「一天的資源」，不是「每場戰鬥的資源」
- 玩家需要規劃「哪一場戰鬥用大招」
- 強力技能消耗大量 MP，需要謹慎使用

#### 消耗品系統（未來擴充）

```rust
pub struct Consumable {
    pub id: String,
    pub name: String,
    pub effect: ConsumableEffect,
    pub quantity: u8,
}

pub enum ConsumableEffect {
    RestoreMp(i32),       // MP 藥水
    RestoreHp(i32),       // HP 藥水
    BuffAccuracy(i32),    // 命中藥水
    // ...
}
```

**設計意圖**：

- MP 藥水很珍貴（稀有、昂貴）
- 玩家需要決定「什麼時候用藥水」
- 跨戰鬥的戰略性資源管理

## 規則系統建議

### ❌ 不要做的事

**1. 不要追求數值平衡（初期）**

原因：

- 先追求「有趣的戰術」，再調整平衡
- 不平衡的機制反而能創造獨特戰術（例如：推下懸崖是「不平衡」的，但很有趣）

**2. 不要做複雜的高度系統**

原因：

- 實作複雜，邏輯難懂
- Terrain + Object 的分離已經足夠
- 掉落傷害計算沒必要，直接「掉落即死」更有戰術意義

### ✅ 應該做的事

**1. 借鑒精神，不抄規則**

從 PF2E/DND 學習的**設計理念**（非具體數值）：

| PF2E/DND 機制 | 設計理念         | 你的實作                    |
| ------------- | ---------------- | --------------------------- |
| 先攻骰        | 回合順序有隨機性 | 已實作：initiative + 1d6    |
| AC 與攻擊骰   | 命中機制         | 已實作：accuracy vs evasion |
| 狀態條件      | 豐富的戰術選擇   | 已有部分（Burn 等），可擴充 |
| 法術位        | 資源管理         | MP 作為一天的限制           |
| 地形規則      | 戰術深度         | Terrain + Object 系統       |

**2. 學習優秀遊戲的設計**

| 遊戲                     | 學習要點           | 應用到你的遊戲             |
| ------------------------ | ------------------ | -------------------------- |
| Divinity: Original Sin 2 | 元素互動、地形戰術 | 狀態協同系統、地形互動     |
| Into the Breach          | 完美信息、位置戰術 | 推擊戰術                   |
| XCOM 2                   | 風險評估、資源管理 | 穩健 vs 賭博技能、MP 管理  |
| 火焰之紋章               | 地形系統、角色養成 | Terrain 影響移動、單位成長 |
| Darkest Dungeon          | 跨戰鬥資源管理     | MP 是一天的限制            |

## 總結

### 你的遊戲核心

**一句話概括**：

> 戰術規劃 + 地形利用 + 資源管理的回合制戰棋 RPG

**核心樂趣循環**：

1. 觀察戰場（地形、敵人、資源）
2. 規劃戰術（技能組合、位置利用）
3. 執行戰術（移動、施法、利用地形）
4. 評估結果（成功 → 成就感 / 失敗 → 調整重試）

**核心理念**：

> **針對性的策略應該能克服隨機性。**

### 必須實作的核心系統

1. ✅ **地形系統**：Pit（深淵）+ Shove（推擊）= 推下懸崖戰術
2. ✅ **命中系統**：百分比系統（不是骰子），可預測的隨機性
3. ✅ **爆擊系統**：crit_rate 欄位，固定 2.0x 倍率
4. ✅ **格擋系統**：Block（機率）與 BlockReduction（減傷）分離
5. ⬜ **資源管理**：MP 是一天的限制，謹慎使用大招
6. ⬜ **風險回報**：穩健技能 vs 賭博技能，玩家可選擇

### 設計哲學

**要做**：

- ✅ 保持簡單，逐步擴充
- ✅ 先追求有趣的戰術，再調整平衡
- ✅ 借鑒優秀遊戲的設計理念（不是抄規則）
- ✅ 確保「穩健戰術」必定存在

**不要做**：

- ❌ 做複雜的高度系統
- ❌ 過早追求完美平衡
