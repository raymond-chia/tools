# 裝備系統設計

## 設計決策

- **不支援動態計算**：裝備變更時預先計算屬性，與現有 `recalc_from_skills` 模式一致
- **可裝卸**：支援裝備/卸除
- **無耐久度**

## 基本結構

- `Equipment`：包含 ID、槽位（武器/護甲/飾品）、賦予的技能列表
- `Unit` 擴展：區分 `learned`（天生）和 `equipped`（裝備）

## 裝備流程

1. 裝備/卸除時更新 `equipped`
2. 收集所有技能：天生 + 裝備賦予
3. 呼叫 `recalc_from_skills()` 重算屬性

## 封印功能

- **封印屬性加成**：現有系統已支援，用負值狀態效果抵消
- **禁用特定技能**：待實作，需新增 `DisableSkill` 效果類型，在技能選擇時檢查

## Tag 設計

新增 `Tag::Equipment` 作為標記標籤（不加入 PRIMARY 分類維度），裝備技能仍保持原有三維分類。
